# План: рекурсивная сборка и прямой ввод выработки изделий

## Контекст

Рабочий в терминале должен отчитаться за готовое изделие одним действием. Система сама проходит по всей цепочке конструктора и выполняет сборку на каждом уровне — от сырья до изделия. Остатки могут уходить в минус (кладовщик может оприходовать сырьё позже).

---

## Фаза 1 — Backend: рекурсивная сборка

Файл: `app/src/services/assembly.service.ts`

### Что сейчас
- `assemble()` работает на один уровень: берёт прямых детей из BOM, проверяет остатки, списывает компоненты, зачисляет изделие.
- Если остатков не хватает — ошибка `AssemblyError("Недостаточно компонентов")`.

### Что нужно
1. Убрать проверку shortages — баланс может уходить в минус.
2. Сделать сборку рекурсивной: если у компонента есть свой BOM — сначала собрать его, потом списать.
3. Вся цепочка — в одной транзакции.

### Алгоритм
1. Рекурсивно обойти BOM целевого изделия, собрать ID всех позиций на всех уровнях.
2. Открыть транзакцию, заблокировать все балансы (FOR UPDATE, ORDER BY item_id ASC).
3. Рекурсивная сборка снизу вверх:
   - Для каждого компонента: если у него есть BOM-дети — рекурсивно собрать нужное количество.
   - Списать компонент, зачислить родителя.
   - Каждый уровень создаёт свою InventoryOperation с уникальным operationKey.
4. Защита от циклов — visited set (архитектура уже запрещает циклы в BOM, но перестраховка).

### Структура кода
- `collectAllBomIds(itemId)` — рекурсивный обход, сбор всех ID.
- `assembleRecursive(tx, itemId, quantity, balanceMap, bomCache, visited)` — внутренняя функция, работает внутри транзакции.
- `assemble(params)` — внешняя функция: собирает ID, открывает транзакцию, блокирует, вызывает рекурсивную сборку.

### Что удаляется
- Массив `shortages` и `AssemblyError("Недостаточно компонентов")` — блокировка по остаткам убирается полностью.

## Фаза 2 — Frontend: терминал

Файл: `app/src/components/terminal/CatalogScreen.tsx`

### Что сейчас
- Клик на изделие → список комплектующих → клик на комплектующую → форма ввода количества.
- Рабочий отчитывается за комплектующие, не за изделие.

### Что нужно
- Клик на изделие → сразу форма ввода количества (как для детали).
- Раздел "Детали" остаётся без изменений — для отдельного учёта заготовок.

### Изменение
- При клике на product в секции "Изделия" — конвертировать в Part и открыть partDetail (аналогично handleBlankClick).
- Промежуточный экран "Комплектующие" для изделий больше не нужен.
