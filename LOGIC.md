# Правила интерфейса склада

## Навигация

- Корень (/) сразу ведёт на склад, без страницы выбора роли
- Навигация "Номенклатура / Остатки / Сборка / Операции" — в хедере, не под ним
- Нет кнопки "На главную" — кладовщик работает только в складе

## Списки и группировка

- Все списки позиций — аккордеон по типам (не плоская таблица)
- Порядок типов слева направо и сверху вниз: от сырья к изделию (Сырьё → Заготовки → Детали → Подсборки → Изделия)
- При поиске аккордеоны раскрываются автоматически

## Карточка позиции

- Показывает состав (из чего делается) и куда входит
- Элементы состава НЕ кликабельные — нельзя проваливаться бесконечно по цепочке
- "Можно собрать" — показывать для позиций с дочерними компонентами, считается из реальных остатков
- Breadcrumbs: Номенклатура / Название позиции

## Визуальные правила

- Шрифты: основной текст — text-sm (14px), вспомогательный — text-xs (12px). Ничего мельче 12px.
- Не использовать прозрачность на тексте (text-muted-foreground/70) — только чистые цвета.
- Бейджи типов: светлый фон + тёмный текст (bg-{color}-100, text-{color}-800, border-{color}-300).
- Светлая тема по умолчанию, dark-варианты не добавлять без запроса.
- Инпуты и кнопки — минимум h-9, text-sm.

## Единообразие

- Placeholder поиска — одинаковый на всех страницах
- Если есть фича (сборка, фильтр и т.д.) — она должна быть отдельным разделом в навигации, а не кнопкой внутри другого раздела
