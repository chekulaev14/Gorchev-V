## Окружение

- Dev server: http://localhost:3000
- БД: postgresql://petrcekulaev@localhost:5432/erp_dev
- psql: /opt/homebrew/Cellar/postgresql@17/17.9/bin/psql

## Перед началом тестирования

1. Определи модуль: конструктор, склад или терминал
2. Прочитай reference файл этого модуля (references/*.md) — там карта UI, элементы, сценарии
3. Если тестируешь складские операции — прочитай также references/db-checks.md

## Workflow тестирования

### Шаг 1 — Открыть браузер

```
mcp__playwright__browser_navigate → http://localhost:3000/{module}
```

### Шаг 2 — Снять snapshot

```
mcp__playwright__browser_snapshot → получить accessibility tree
```

Snapshot — основной способ "видеть" страницу. Это текстовое описание всех элементов на экране: кнопки, текст, поля ввода, ссылки — с их текстом и состоянием. По snapshot ты понимаешь что где находится и куда кликать. Делай snapshot после каждого значимого действия.

### Шаг 3 — Выполнить сценарий

Используй инструменты MCP Playwright:
- `browser_click` — клик по элементу (ref из snapshot)
- `browser_type` — ввод текста
- `browser_fill_form` — заполнение формы
- `browser_select_option` — выбор из dropdown
- `browser_press_key` — нажатие клавиши (Escape, Enter, Tab)
- `browser_wait_for` — ожидание элемента

После каждого действия → snapshot чтобы увидеть результат.

### Шаг 4 — Проверить console

```
mcp__playwright__browser_console_messages
```

Собрать ошибки. Любая ошибка в console — потенциальный баг.

### Шаг 5 — Проверить БД (для операций с данными)

Toast "успешно" — не доказательство. Проверяй данные в БД через psql.
См. references/db-checks.md для типовых запросов.

### Шаг 6 — Отчёт

Кратко сообщи:
- Что проверено (какие сценарии)
- Console ошибки (если есть)
- Состояние данных в БД (если проверялось)
- Найденные баги

## Правила

- Скриншоты — только с явного разрешения пользователя. По умолчанию используй snapshot.
- Toast ≠ доказательство. Проверяй данные в БД.
- Если элемент не найден в snapshot — страница могла не загрузиться. Подожди через browser_wait_for.
- Не добавляй sleep/паузы без причины. Playwright сам ждёт элементы.
- При ошибке навигации — проверь что dev server запущен (localhost:3000).
