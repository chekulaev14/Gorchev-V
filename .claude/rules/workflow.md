ОБЯЗАТЕЛЬНО перед началом любой задачи (до чтения кода, до анализа, до любых действий):

1. Проверить подходящий Superpowers скилл через Skill tool. Даже если кажется что скилл не нужен — проверить. Если подходит хотя бы на 1% — вызвать.
2. Создать beads issue (`bd create`) для задачи. Без issue не начинать работу.
3. Перевести issue в in_progress (`bd update <id> --status=in_progress`).

Порядок: Superpowers скилл → beads issue → работа.

Это НЕ опционально. Пропуск этих шагов = нарушение workflow.
