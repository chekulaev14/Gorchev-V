# Gorchev-V — План реализации

## Стек
- Next.js + TypeScript
- Tailwind CSS + shadcn/ui
- PostgreSQL + Prisma ORM
- Фото — локальное хранение

## Модули

### Терминал рабочего
- [x] демо — экран PIN, выбор изделия/детали с фото, галерея, ввод количества
- [x] демо — автовыход по таймауту бездействия
- [x] БД — сохранение выработки в базу (с фиксацией даты и времени отправки)
- [ ] история выработки рабочего в личном кабинете

### Номенклатура
- [x] демо — 93 позиции: сырьё, заготовки, детали, подсборки, изделия
- [x] демо — многоуровневый BOM (сырьё → заготовка → деталь → подсборка → изделие)
- [x] демо — просмотр справочника с фильтрами по типу и категории, поиск
- [x] демо — карточка позиции: состав (из чего делается) и куда входит
- [x] CRUD 1 — кнопка "Режим редактирования" в хедере + контекст EditMode
- [x] CRUD 2 — редактирование полей позиции в карточке (название, описание, тип, единица, категория, расценка)
- [x] CRUD 3 — добавление новой позиции
- [x] CRUD 4 — удаление позиции (soft delete, корзина «Удалённые» с восстановлением, BOM-связи сохраняются)
- [x] CRUD 5 — редактирование BOM (добавить/убрать компонент, изменить количество)
- [x] Конструктор изделия — пошаговое массовое создание цепочки (изделие → подсборки → детали → заготовки → сырьё)
- [ ] загрузка фото для позиций
- [x] БД — перенос номенклатуры в PostgreSQL + Prisma (требует проверки)

## Критерии Go/No-Go

### Продолжаем если:
- [ ] Кладовщик ведёт учёт через систему без помощи
- [ ] Рабочие пользуются терминалом каждую смену
- [ ] Директор перестала спрашивать "а сколько у нас на складе" — смотрит сама

### Меняем курс если:
- [ ] Кладовщик возвращается к тетрадке — интерфейс слишком сложный
- [ ] Рабочие не пользуются терминалом — неудобно или медленно
