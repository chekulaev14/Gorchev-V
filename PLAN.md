# ERP — План реализации

## Стек
- Next.js + TypeScript
- Tailwind CSS + shadcn/ui
- PostgreSQL + Prisma ORM
- Фото — локальное хранение

## Модули

- [ ] рабочий видит свой заказ в терминале, отмечает факт выработки
- [ ] при завершении: списание по факту, освобождение неиспользованного сырья (план 50, сделал 47 → 3 шт сырья возвращается)

Учитывает переплетение цепочек (разные изделия из одного сырья)

## Критерии Go/No-Go

### Продолжаем если:
- [ ] Кладовщик ведёт учёт через систему без помощи
- [ ] Рабочие пользуются терминалом каждую смену
- [ ] Директор перестала спрашивать "а сколько у нас на складе" — смотрит сама

### Меняем курс если:
- [ ] Кладовщик возвращается к тетрадке — интерфейс слишком сложный
- [ ] Рабочие не пользуются терминалом — неудобно или медленно
